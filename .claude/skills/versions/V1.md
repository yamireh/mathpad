# MathPad — V1 Scope

## Goal

Ship the **first production version** to the App Store **with the Operations monetization live**. Free download. **Addition is free** (full features). **Subtraction + Multiplication + Division + Mix unlock together for $9.99** — the "Operations bundle" IAP from the [[pricing]] skill. The other modules (Clock / Shapes / Axis) remain **Coming Soon** and will be paid later — *no rush*. V1 validates both the product and the core purchase in the wild.

> Decision (2026-06-03): V1 adopts the pricing model. The $9.99 Operations bundle is a **live in-app purchase at launch**, not deferred. This supersedes the earlier "free, no IAP in V1" plan.

## What's IN V1

- **Operations module — Addition free, rest paid**
  - **Addition** — free: full features, all settings, history, score, review
  - **Subtraction · Multiplication · Division · Mix** — gated behind the **$9.99 Operations bundle** IAP
  - All settings: digit range, question count, timer, carrying/borrowing/regrouping/remainder/decimal/negative-answer modes
  - Handwriting recognition + scratch work (carry boxes, borrow arrows, division draft grid)
  - Practice flow → Score → Review/Edit
- **Monetization (Operations tier only)**
  - StoreKit purchase of the Operations bundle ($9.99 non-consumable)
  - **Restore Purchases**
  - **Entitlement check** — single source of truth (on-device + StoreKit receipt), queried wherever a paid operation is reached
  - **Lock / unlock UI** on the paid operations (Try-Free framing, never "BUY NOW") + an upgrade/store surface (per pricing §3)
  - Refund-safe purchase-confirmation copy
  - Family Sharing ON for the IAP (recommended; final confirm pending)
- **History** of completed sessions (local, offline)
- **Settings** (per-operation, persisted locally)
- **All four operation accent colors** (blue / coral / purple / teal) plus Mix navy
- **i18n: English** (Arabic and Spanish post-V1)
- **iOS + iPad** (Android post-V1)
- **Fully offline** — no backend, no accounts, no analytics, no ads (per SPEC). StoreKit's local receipt is the only gate.

## What's NOT in V1 (deferred)

- **Clock / Shapes / Axis modules** — shown as "Coming Soon" placeholders. Their IAPs ($7.99 each) and free slices are deferred until each module is built.
- **MathPad Complete bundle ($24.99)** — deferred; it only makes sense once the other modules exist. **V1 ships exactly one IAP: the Operations bundle.**
- **30-day usage nudge** — optional polish, not a launch gate (the free-Addition → buy-Operations nudge from pricing §3.4 can land in a V1.x update).
- **Localizations** beyond English
- **Android build**

## Out of scope for V1 (explicit non-goals)

- Clock / Shapes / Axis **module builds** and their IAPs — separate versions
- **Complete bundle** SKU — deferred until other modules exist
- Subscriptions / free trials — rejected per pricing §2 and §5 (offline + no-backend)
- 30-day nudge — optional, post-launch
- A/B tests, feature flags, remote config — disallowed by the no-backend constraint
- Android build; additional languages

## Rules for working in V1 scope

1. **The Operations IAP IS in scope.** Build entitlement gating cleanly: Addition free, Sub/Mult/Div/Mix behind one $9.99 non-consumable, with Restore Purchases. Entitlement is a single source of truth queried everywhere — don't scatter purchase checks.
2. **Don't build the other modules' pricing yet.** No Clock/Shapes/Axis IAPs, no Complete bundle SKU — those are post-V1.
3. **Follow the cross-sell rules** (pricing §3): lock UI uses Try-Free framing; never upsell mid-session, on Score, or on Review/Edit; no countdown/interstitial pressure.
4. **Don't ship dev affordances.** Auto-Solve, debug toggles, recognition-test screens — gated off in production builds.
5. **Treat Coming Soon as a real UX surface.** Clock / Shapes / Axis placeholders are the first thing parents see after Operations — keep them polished, not TODOs.
